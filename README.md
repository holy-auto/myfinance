# 自社専用の経理ソフト構築ガイド（MVP）

このリポジトリは、**自社業務に最適化した経理ソフト**を段階的に構築するための最小実装（MVP）設計をまとめたものです。

## 1. 目的

- 市販会計ソフトで吸収しづらい自社固有業務（承認フロー・部門配賦・取引分類）を内製で実現
- 手作業の仕訳入力・照合・月次締めを自動化
- 監査対応のためのログ・証跡を標準実装

## 2. MVPスコープ（最初の3か月）

### 必須機能

1. 仕訳管理
   - 手動入力
   - CSV取込
   - 勘定科目・補助科目・部門タグ付与
2. 証憑管理
   - 請求書・領収書アップロード
   - 仕訳との関連付け
3. 月次締め
   - 期間ロック
   - 締め後の修正は取消仕訳で対応
4. レポート
   - 試算表
   - 総勘定元帳
   - 部門別損益（簡易）
5. 権限管理
   - 経理担当 / 承認者 / 閲覧者

### MVPで後回しにする機能

- 固定資産管理
- 連結会計
- 複数通貨・為替再評価
- 高度な予実管理

## 3. 推奨アーキテクチャ

- フロントエンド: Next.js + TypeScript
- API: FastAPI または Node.js (NestJS)
- DB: PostgreSQL
- ファイル保管: S3互換ストレージ
- 認証: SSO (OIDC/SAML) + RBAC
- アカウント管理: 社員名簿（IdP/HR）連携のプロビジョニング（入社時自動作成・退職時自動無効化）
- 監査ログ: 追記専用テーブル + 改ざん検知ハッシュ

## 4. データモデル（最小）

- `accounts`（勘定科目）
- `journal_entries`（伝票ヘッダ）
- `journal_lines`（借方/貸方明細）
- `attachments`（証憑）
- `entry_attachments`（伝票と証憑の中間）
- `closing_periods`（締め期間）
- `users` / `roles` / `user_roles`
- `audit_logs`

## 5. 業務フロー（基本）

1. 証憑アップロード
2. 仕訳起票（AI補助分類は将来拡張）
3. 承認ワークフロー
4. 月次締め（ロック）
5. レポート出力

## 6. 非機能要件

- 可用性: 99.9%（業務時間帯）
- 性能: 一覧表示 2秒以内（1万件規模）
- セキュリティ: 2FA、IP制限、暗号化（保存時/通信時）
- 閲覧制御: 「自社の人間のみ閲覧可」を強制（社外メールドメインのサインアップ禁止・未承認アカウントのアクセス拒否）
- アカウントポリシー: 招待制のみ、管理者承認必須、最小権限付与、90日ごとの権限棚卸し
- 監査性: 操作ログ保存 7年
- バックアップ: 日次フル + 15分毎WAL

## 6.1 アカウント運用ルール（社外閲覧防止）

1. ユーザー作成経路を **招待制** のみに限定（自己登録は無効化）
2. 招待可能者は「システム管理者」のみ
3. ログイン許可条件
   - 会社のIdPに存在すること
   - 有効な雇用状態であること（休職/退職は停止）
   - 必須MFAを通過していること
4. 初回ログイン時に利用規約・監査ポリシー同意
5. 退職・委託終了時は即時失効（セッション強制切断）
6. 四半期ごとにアカウント棚卸しを実施し、不要権限を削除

## 7. 実装ロードマップ

### Phase 0（1〜2週）
- 要件定義（業務ヒアリング）
- 勘定科目体系・承認フロー確定
- 画面モック作成

### Phase 1（3〜6週）
- 認証/権限
- 仕訳入力・CSV取込
- 証憑アップロード

### Phase 2（7〜10週）
- 承認フロー
- 月次締め
- 試算表・元帳

### Phase 3（11〜12週）
- 運用テスト
- 既存データ移行
- 本番リリース

## 8. 受け入れ基準（例）

- 代表的な仕訳パターン20件を誤差なく登録可能
- 月次締め後に過去仕訳の直接更新ができない
- 試算表と元帳残高が一致する
- 主要操作が監査ログに記録される
- 自社外アカウント（未招待・社外ドメイン・退職者）でログインできない

## 9. 次のアクション

1. 現行業務フロー（現場のExcel/紙運用）を洗い出す
2. 必須帳票を優先順位付けする
3. MVP対象を確定し、画面遷移図を作る
4. 2週間単位の開発スプリントを開始
