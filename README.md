# 自社専用の経理ソフト構築ガイド（MVP）

このリポジトリは、**自社業務に最適化した経理ソフト**を段階的に構築するための最小実装（MVP）設計をまとめたものです。

## 1. 目的

- 市販会計ソフトで吸収しづらい自社固有業務（承認フロー・部門配賦・取引分類）を内製で実現
- 手作業の仕訳入力・照合・月次締めを自動化
- 監査対応のためのログ・証跡を標準実装

## 2. MVPスコープ（最初の3か月）

### 必須機能

1. 仕訳管理
   - 手動入力
   - CSV取込
   - 勘定科目・補助科目・部門タグ付与
   - 自動仕訳（ルール適用）と学習による候補精度向上
2. 証憑管理
   - 請求書・領収書アップロード
   - 仕訳との関連付け
   - OCRで写真撮影後に金額・日付・取引先を抽出
   - 抽出結果から勘定科目/税区分/部門を自動割り振り（最終承認は人手）
   - 証憑添付・長期保存（監査証跡付き）
3. 月次締め
   - 期間ロック
   - 締め後の修正は取消仕訳で対応
4. レポート
   - 仕訳帳
   - 試算表
   - 損益計算書（PL）
   - 貸借対照表（BS）
   - 総勘定元帳
   - 部門別損益（簡易）
   - 事業別PL/売上・粗利・販管費の可視化
5. 権限管理
   - 経理担当 / 承認者 / 閲覧者
6. 経営数値管理（ダッシュボード）
   - 売上、粗利、営業利益、キャッシュ残高、売掛/買掛回転日数
   - 予実比較（月次・四半期）
   - KPIアラート（閾値超過時に通知）
7. 外部連携/API
   - 銀行・クレジットカード・電子マネー明細連携（自動取得）
   - 会計データ参照/登録のための外部API（社内システム連携）

## 2.2 銀行連携の実現方針

- **対応可能**。初期は「連携サービス経由 + CSV取込」を併用し、段階的に自動同期へ移行
- 取得対象: 入出金明細、振込手数料、口座残高（取得可能な範囲）
- 同期方式: 日次バッチ + 手動再同期（障害時リカバリ）
- 突合処理: 既存仕訳との自動照合（同日/近似金額/取引先名）
- セキュリティ: アクセストークン暗号化保存、連携解除時の即時無効化
- 例外時運用: 連携失敗時はCSVインポートで業務継続

## 2.1 ご要望機能の対応状況

- 帳簿・決算書作成（仕訳帳 / 総勘定元帳 / 試算表 / PL / BS）: **対応**
- 銀行・クレカ・電子マネー連携: **対応**
- 自動仕訳・仕訳ルール学習: **対応**
- 領収書・請求書のOCR取り込み: **対応**
- 証憑添付・保存: **対応**
- インボイス制度・電子帳簿保存法対応: **対応（要件明記）**
- 外部連携/API: **対応**

### MVPで後回しにする機能

- 固定資産管理
- 連結会計
- 複数通貨・為替再評価
- 高度な予実管理

## 3. 推奨アーキテクチャ

- フロントエンド: Next.js + TypeScript
- API: FastAPI または Node.js (NestJS)
- DB: PostgreSQL
- ファイル保管: S3互換ストレージ
- 認証: SSO (OIDC/SAML) + RBAC
- アカウント管理: 社員名簿（IdP/HR）連携のプロビジョニング（入社時自動作成・退職時自動無効化）
- 監査ログ: 追記専用テーブル + 改ざん検知ハッシュ
- OCR/分類基盤: OCRエンジン + ルールベース + 学習済み分類モデル（信頼度付き提案）
- 金融データ連携: Open Banking/金融連携サービス経由で明細同期
- 連携API: REST API + Webhook（仕訳登録、証憑連携、承認イベント通知）

## 4. データモデル（最小）

- `accounts`（勘定科目）
- `journal_entries`（伝票ヘッダ）
- `journal_lines`（借方/貸方明細）
- `attachments`（証憑）
- `entry_attachments`（伝票と証憑の中間）
- `closing_periods`（締め期間）
- `business_units`（事業・サービス区分）
- `projects`（案件/プロジェクト）
- `users` / `roles` / `user_roles`
- `audit_logs`
- `kpi_snapshots`（経営指標の時系列スナップショット）
- `budget_plans`（予算計画）

## 5. 業務フロー（基本）

1. 証憑アップロード
2. OCR実行（写真/画像/PDFから文字情報を抽出）
3. 自動割り振り（勘定科目・税区分・部門を提案）
4. 経理担当が確認・修正して仕訳起票
5. 承認ワークフロー
6. 月次締め（ロック）
7. レポート出力

## 5.1 OCR自動割り振りルール（初期）

- 取引先名 + 摘要キーワードで勘定科目候補を優先表示
- 税率・税込/税抜の表記から税区分を推定
- 店舗/プロジェクトコードを検出した場合は部門候補を提示
- 信頼度が閾値未満（例: 0.8）の場合は「要確認」に自動振り分け
- 同じ取引先・同金額帯・同用途の履歴を学習し次回提案に反映

## 5.2 会社経営に必要な数字管理（管理会計）

- 月次の経営ダッシュボードを自動生成（売上/粗利/営業利益/EBITDA/運転資金）
- 予実差異を「金額」と「率」で表示し、前年差も併記
- 資金繰り管理として、入出金予定・キャッシュ残高見込みを表示
- 重要KPI（例: 粗利率、広告費率、LTV/CAC）を閾値監視
- 会計データ確定後にKPIスナップショットを自動保存

## 5.4 日次・週次・月次の運用サイクル

- 日次: 銀行/クレカ明細同期、未処理証憑チェック、資金繰り見込み更新
- 週次: 予実ギャップ確認、部門/事業別KPIレビュー、未承認伝票の解消
- 月次: 月次締め、試算表/PL/BS確定、監査ログと証憑突合、KPIスナップショット固定
- すべてのサイクル結果を履歴保存し、過去期間との比較分析を可能にする

## 5.5 資金調達向けの数値管理（銀行・経営者提出用）

- 銀行提出向け: 月次試算表、PL、BS、資金繰り表、借入返済予定表を出力
- 経営者向け: 事業別売上推移、粗利率、EBITDA、運転資金推移、キャッシュランウェイ
- 投資家/金融機関向け補足指標: ARR/MRR、churn、LTV/CAC（該当ビジネスのみ）
- 指標は「確定値」と「予測値」を分離して保存し、提出資料の再現性を担保
- 提出用エクスポート: PDF/Excel/CSV（出力時点と作成者を監査ログに記録）

## 5.6 事業内容ごとの数字管理（セグメント管理）

- すべての仕訳に `business_unit`（事業区分）と任意で `project` を付与
- 共通費は配賦ルール（売上比/人員比/固定配賦）で事業別に配賦
- 事業別にPL（売上・粗利・営業利益）を出力
- 事業別のKPI（例: SaaSならMRR/churn、小売なら客単価/来店数）を定義可能
- 「全社」「事業別」「プロジェクト別」で同一指標をドリルダウン表示

## 6. 非機能要件

- 可用性: 99.9%（業務時間帯）
- 性能: 一覧表示 2秒以内（1万件規模）
- セキュリティ: 2FA、IP制限、暗号化（保存時/通信時）
- 閲覧制御: 「自社の人間のみ閲覧可」を強制（社外メールドメインのサインアップ禁止・未承認アカウントのアクセス拒否）
- アカウントポリシー: 招待制のみ、管理者承認必須、最小権限付与、90日ごとの権限棚卸し
- 法令対応: インボイス制度・電子帳簿保存法の要件（検索性、真実性、見読性）を満たす
- 監査性: 操作ログ保存 7年
- バックアップ: 日次フル + 15分毎WAL

## 6.1 アカウント運用ルール（社外閲覧防止）

1. ユーザー作成経路を **招待制** のみに限定（自己登録は無効化）
2. 招待可能者は「システム管理者」のみ
3. ログイン許可条件
   - 会社のIdPに存在すること
   - 有効な雇用状態であること（休職/退職は停止）
   - 必須MFAを通過していること
4. 初回ログイン時に利用規約・監査ポリシー同意
5. 退職・委託終了時は即時失効（セッション強制切断）
6. 四半期ごとにアカウント棚卸しを実施し、不要権限を削除

## 7. 実装ロードマップ

### Phase 0（1〜2週）
- 要件定義（業務ヒアリング）
- 勘定科目体系・承認フロー確定
- 画面モック作成

### Phase 1（3〜6週）
- 認証/権限
- 仕訳入力・CSV取込
- 証憑アップロード
- 銀行明細CSV取込（先行リリース）

### Phase 2（7〜10週）
- 承認フロー
- 月次締め
- 試算表・元帳
- 銀行/クレカ/電子マネー明細の自動同期
- 日次/週次バッチ（同期・差異検知・アラート）の定期実行

### Phase 3（11〜12週）
- 運用テスト
- 既存データ移行
- 本番リリース
- 銀行・経営者提出向けレポートテンプレート確定

## 8. 受け入れ基準（例）

- 代表的な仕訳パターン20件を誤差なく登録可能
- 月次締め後に過去仕訳の直接更新ができない
- 試算表と元帳残高が一致する
- 主要操作が監査ログに記録される
- 自社外アカウント（未招待・社外ドメイン・退職者）でログインできない
- OCR読取項目（取引日/金額/取引先）の抽出精度が目標値を満たす（例: F1 0.90以上）
- 自動割り振り提案のうち、上位候補での一次採択率が70%以上
- 事業区分を付与した仕訳から事業別PLが月次で自動作成される
- 経営ダッシュボードの主要KPIが月次締め後24時間以内に更新される
- 仕訳帳・総勘定元帳・試算表・PL・BSを同一締め期間で整合性を保って出力できる
- 銀行/クレカ/電子マネー明細の自動取込で、手入力件数を基準値から50%以上削減
- 電子帳簿保存法の検索要件（取引日・金額・取引先）で証憑検索ができる
- 日次/週次/月次の各ジョブ実行結果（成功/失敗、実行者、実行時刻）が追跡できる
- 銀行提出用資料（試算表・PL・BS・資金繰り表）を期間指定で再出力できる
